# JustMC JVM
JustMC JVM (*далее - JJVM*) - Транслятор байт-кода JVM в блочный код [JustMC](https://justwiki.gitbook.io/wiki/creative/about).\
*(А виртуальная машина прячется на самом JustMC, где есть интерпретатор блоков кода)*

### Как это работает
Для работы с JJVM нужно:
1. Создать проект Java или Kotlin, выбрав Gradle и Kotlin DSL
2. Скачать [JJVM и JJDK](https://github.com/unidok/justmc-jvm/releases)
3. [Настроить конфигурацию сборки](#настройка-конфигурации-сборки)
4. Выполнить задачу `buildModule` или `uploadModule` 

### Настройка конфигурации сборки
```kts
fun a(): Int {}
```

### Примитивы
Не гарантируется правильная работа примитивов (`byte`, `short`, `char`, `int`, `long`, `float`, `double`), ведь на JustMC нет этих типов данных.
Все перечисленные типы будут восприняты как обычное число.
Операции превращения из одного примитивного типа в другой будут проигнорированы,
за исключением превращения *типа с плавающей точкой* (`float`, `double`) в *целочисленный тип* (`byte`, `short`, `char`, `int`, `long`) - в этом случае будет добавлена операция *округления в меньшую сторону*.

### Объекты
В JVM доступ к объекту происходит по ссылке. Ссылка, как и другие примитивы, будет воспринята как число.\
Специальная константа `null` эквивалентна числу 0.
#### Создание объекта
При создании объекта выделяется память для него, а затем возвращается ссылка на этот объект.\
В JJVM выделение памяти самое примитивное: 
```java
long ADR = 0;
long NEW(String clazz) {
  long address = ++ADR;
  GAME_VAR(Long.toString(address)) = CREATE_MAP("class", clazz);
  return address;
}
```

### Java
Практически ничего из Java нельзя использовать, потому что она вырезана из-за её большого размера.
Поэтому у вас не получится странслировать любой джарник и запустить его. Странслировать-то получится, но смысла от этого нет,
потому что без JJDK оно никак не сможет взаимодействовать с миром на JustMC.
Выходит, что любой полезный код надо будет писать в интеропе с JJDK, то есть специально под JustMC.
То же самое касается kotlin-stdlib и любых других библиотек, использующих Java (их 100%)
